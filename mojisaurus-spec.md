# もじざうるす（MojiSaurus）— 国語あそび 仕様書 v2

> **この文書について**
> ひらがな・カタカナが**読める5歳児**向けの国語学習ゲームを、別のAI（Claude Code）が正確に実装するための仕様書です。
> 実装者は本書の「用語」「受け入れ条件（Definition of Done）」に厳密に従うこと。迷う箇所は *推奨値* を採用し、勝手な機能追加はしないこと。
> タイトル「もじざうるす」は仮称。変更可。

> **v2での変更点（前提レベルの引き上げ）**
> 対象児は **ひらがなは全て読め、カタカナも概ね読める**。よって「文字を読む/探す」基礎ゲームは廃止し、**就学前準備の次段階**へ振り直した：
> ①**書き**（ひらがな→カタカナ、筆順つき） ②**カタカナ仕上げ**（まぎらわしい字・ひらカタ対応） ③**特殊音**（濁音・半濁音・拗音・促音・長音） ④**助詞**（は・を・へ） ⑤**語彙・ことばあそび**（しりとり・仲間分け・反対ことば） ⑥**短い文を読む**（読解の芽）。

## 0. サマリ（最初にこれだけ読めば全体がわかる）

- **対象**: 5歳（年長・就学前）。**ひらがな読了、カタカナ概ね読了**。→ 読む力を前提にした出題を積極的に使う。
- **学習目標**: (1) ひらがな・カタカナを**正しい筆順で書く**、(2) カタカナの**まぎらわしい字**を確実に、(3) **濁音・半濁音・拗音・促音・長音**の読み書き、(4) **助詞は/を/へ**の読み・使い分け、(5) **語彙**を増やす、(6) **短い文**を読んで意味をとる。
- **形式**: 短いミニゲームの集合（1ゲーム 40〜90秒）。数本束ねた「レッスン」1回=約4〜6分。
- **動機付け**: クリアで **たまご** が育ち、孵ると **恐竜・昆虫・おばけ** の仲間が増える → **ずかん** を集める。仲間の名前は**ひらがな/カタカナ両表示**にでき、読みの副次学習も兼ねる。
- **設計思想**: こどもちゃれんじ準拠 = ①キャラが案内 ②常に肯定（叱らない） ③音声は補助として常備 ④短サイクル ⑤ごほうび即時。
- **技術**: Canvas 2D + ES Modules + 独自ビルド（既存プロジェクト *Bramble Boom* と同じ規約に合わせる）。外部フレームワーク不使用。
- **端末**: タブレット最適化＋レスポンシブ（指・スタイラス操作前提）。
- **保存**: `localStorage`（サーバー不要・完全オフライン動作可）。

## 1. デザイン原則（こどもちゃれんじ参考・実装判断の基準）

実装で迷ったら、この原則に照らして決めること。

1. **読める力を前提に、それを活かす**：文字・短文を読ませる出題を積極採用。ただし音声ガイドは**補助として常備**（説明・出題・褒めは音声でも行い、詰まらせない）。長い説明文は出さない。
2. **常に肯定する**：不正解でも否定音を強く出さない。「おしい！ もういちど」と優しく促し、正解に導く。失敗回数を子どもに表示しない。
3. **正解体験を必ず作る**：数回まちがえたら選択肢を減らす／ヒントを出し、**最終的に必ず正解できる**導線にする。
4. **短く・即報酬**：1操作ごとに小さな演出（星が飛ぶ・音・キャラが喜ぶ）。
5. **キャラが主役**：ナビキャラが全編で話しかけ、褒め、しりとり等では**対戦相手**にもなる。
6. **誤操作に強い**：タップ領域は大きく（最小サイズ厳守）。設定変更・終了は保護者ゲートの奥に隠す。
7. **中断に強い**：いつ閉じても進捗が保存され、次回は続きから。最後までやらせる圧をかけない。
8. **易しすぎない**：読める子が退屈しないよう、書き・特殊音・助詞・語彙など**考える要素**を混ぜ、達成感を出す。

## 2. 世界観・キャラクター

- **舞台**：「もじのしま」。恐竜・昆虫・おばけが仲良く暮らす島。
- **ナビゲーターキャラ**：小さな恐竜 **「ガオ」**（仮名）。プレイヤーの相棒であり、しりとり等では対戦相手。
  - 声：Web Speech API（後述）。優しく、テンションは高すぎない。
  - 表情差分：ふつう／うれしい／おうえん／びっくり／こまった（しりとりで負けそう）の5種。
- **コレクション対象（仲間）**：3系統。
  - 🦖 **きょうりゅう**（ティラノ、トリケラ、プテラ… 丸くデフォルメ）
  - 🐛 **むし**（かぶとむし、ちょうちょ、てんとうむし、ばった…）
  - 👻 **おばけ**（まるおばけ、ぼうしおばけ… 怖くない・かわいい）
  - 仲間の名前は **ひらがな表記とカタカナ表記の両方**を持ち、ずかんで切替表示（例：`てぃらの` / `ティラノ`）。カタカナ読みの練習を兼ねる。
- **トーン**：明るくポップ。怖い表現・暗い配色は避ける。

> **アセット注記**：キャラ画像が未用意の段階では **図形（Canvas描画）＋絵文字フォント** で暫定表示可。差し替え可能なようアセットIDで参照する（`assets/creatures/dino_01.png` 等を設定ファイルで管理）。

## 4. ミニゲーム仕様（本書の中核）

各ゲームは独立モジュール。共通インターフェース（§8.3）に従う。**Pillar A/D をP1で実装**、以降を段階追加（§Phase表）。

### 共通の演出ルール（全ゲーム共通）

- **出題時**：ガオが問題を音声で言う。関連する文字・絵は画面にも出す。
- **正解時**：対象がジャンプ／光る＋「ピンポン♪」＋星が飛んでゲージへ。ガオが「うれしい」表情。
- **不正解時**：選択が軽く揺れる＋控えめな「ん？」音。正解は消さず再挑戦可。
- **ヒント段階**：同一問題で2回まちがえたら正解を淡く光らせ、3回目は誤答を薄くする。→ **必ず正解できる**。
- **1問の目安**：4〜8秒。1ゲーム=5〜8問。

### Pillar A ── かく（書き・筆順つき）

#### 【W1】なぞりがき・ひらがな
- **ねらい**：読める字を**正しい筆順で書く**運動記憶（読み≠書き）。
- **画面**：中央に大きく1文字の**お手本（薄いグレー）**。ガオが上で応援。
- **手順**：
  1. **筆順アニメ再生**：手本の線が1画ずつ順に描かれる（+「いち、に…」音声）。動く指が軌跡を見せる。
  2. **なぞりターン**：子どもが1画ずつ指でなぞる。現在なぞる画を強調表示。
- **なぞり判定**：§7の寛容アルゴリズム。逆走・大外れのみ「もういちど、じゅんばんに」と促す。
- **完了**：全画なぞり終了で文字が色づき＋キラキラ＋星。
- **難易度パラメータ**：`R`（許容半径）、お手本の濃さ、筆順ガイド常時ON/OFF。
- **禁止**：手書き文字認識（字形の正誤判定）はしない。「順番通りになぞれたか」だけを見る。
- **DoD**：筆順アニメ・画ごとのなぞり判定・寛容な許容・完了演出。

#### 【W2】なぞりがき・カタカナ（P2）
- W1と同機構でカタカナ。データ差し替えのみで実現できる設計にする。

#### 【W3】ひつじゅんクイズ（発展・P4）
- **ねらい**：筆順の理解確認（なぞりより思考的）。
- **画面**：途中まで書かれた文字＋「つぎは どのせん？」。次に書く一画の候補を2〜3本提示。
- **操作**：正しい次の一画を選ぶ。
- **DoD**：各文字の画順データから「正しい次の一画」を提示・判定。

### Pillar B ── カタカナ しあげ

#### 【K1】にてるカタカナ（P2）
- **ねらい**：まぎらわしいカタカナの識別。
- **画面**：似た2〜3字を並べる（`シ/ツ`、`ソ/ン`、`ク/ワ`、`ノ/メ/ヌ`、`ス/ヌ`、`ア/マ`）。
- **出題**：ガオが音で指定「『シ』は どれ？」。
- **操作**：正しい方をタップ。
- **データ**：§6 `confusableSets`（カタカナ版）。
- **DoD**：まぎらわしいセットから出題・識別判定・ヒント。

#### 【K2】ひらがな⇔カタカナ（P2）
- **ねらい**：同じ音のひらがな/カタカナ対応（`き`↔`キ`）。
- **画面**：神経衰弱風、または左右マッチング。ひらがなカードとカタカナカードを対にする。
- **操作**：同じ音のペアを選ぶ／めくって合わせる。
- **DoD**：ペア成立判定・不一致は戻す・完成演出。

### Pillar C ── とくべつな おと（濁音・半濁音・拗音・促音・長音・助詞）

#### 【S1】てんてん・まる（濁音・半濁音）（P3）
- **ねらい**：清音＋濁点/半濁点で音が変わる理解（`か`＋てんてん=`が`、`は`＋まる=`ぱ`）。
- **画面**：中央に清音1字。横に「てんてん(゛)」「まる(゜)」のスタンプ。
- **出題**：ガオが目標音を言う「『が』を つくろう」。
- **操作**：正しいスタンプを文字にタップ（ドラッグ）して貼る → 音が変わって再生。
- **難易度**：清音→目標が濁音/半濁音のどちらか判断。ダミー清音を混ぜる。
- **DoD**：スタンプ貼付インタラクション・変化音再生・正誤判定。

#### 【S2】ちいさいもじ（拗音・促音・長音）（P3）
- **ねらい**：小さい `ゃゅょ`・小さい `っ`・伸ばす `ー` の読み書き。
- **画面と出題（サブモード）**：
  - **拗音づくり**：`き`＋`ゃ`→`きゃ`。大きい字＋小さい字を並べて目標語を作る。
  - **促音ききわけ**：音声「きって」／「きて」を聞き、小さい`っ`が**ある/ない**を選ぶ。
  - **長音**：「おかあさん」等、伸ばす音を含む語を正しく作る/選ぶ。
- **DoD**：各サブモードの生成/選択・音声・判定。まず拗音づくりを実装、他は追加可。

#### 【B2】じょしパズル（助詞 は/を/へ）（P4）
- **ねらい**：助詞 `は`(=わ)・`を`(=お)・`へ`(=え) の使い分けと読み。
- **画面**：短文の助詞部分が空欄「ぼく（　）はしる」。候補 `は/を/へ` を提示。
- **出題**：ガオが文を読み上げ、空欄で「なにが はいる？」。
- **操作**：正しい助詞を選ぶ。正解時、文全体を**正しい読み**で読み上げ（`は`→「わ」等）。
- **注記**：難度高め。P4。まず`は`だけ→3種へ拡張。
- **DoD**：文＋空欄＋助詞選択・正誤・正しい読み上げ。

### Pillar D ── ことば・ことばあそび（語彙／音韻）

#### 【L1】しりとり（P1）
- **ねらい**：語彙拡張＋語尾音の意識。
- **画面**：ガオと交互。中央に現在の語（絵＋文字）。下に次の候補カード3枚（絵＋語）。
- **ルール**：直前の語の**最後の音**で始まる語を選ぶ。`ん`で終わる語は選べない（選ぶと「あーっ、ん がついちゃう！」で優しく無効）。長音`ー`・小さい`っ`で終わる場合の扱いも定義（`ー`は直前の母音、`っ`は避ける）。
- **操作**：正しい語カードをタップ → ガオが次の語で返す → 繰り返し。数ターンで1ゲーム。
- **拡張**：慣れたらカードから選ぶ→バラのかなで**自分で作る**モードへ。
- **DoD**：語尾音マッチ判定・`ん`回避・ガオの返し（辞書から選択）・数ターン継続。

#### 【L2】なかまあつめ（カテゴリ分け）（P1）
- **ねらい**：語彙とカテゴリ概念。
- **画面**：上に2〜3個のかご（例 `たべもの`/`のりもの`/`どうぶつ`）。中央に絵＋語カードが数枚。
- **操作**：各カードを正しいかごへドラッグ（またはタップ→かご選択）。
- **データ**：§6 `categories.json`。
- **DoD**：カテゴリ判定・かご振り分け・全問正解で完成演出。

#### 【L3】はんたいことば（反対語）（P3）
- **ねらい**：反対語の語彙（`おおきい`⇔`ちいさい`、`たかい`⇔`ひくい`…）。
- **画面**：お題の語/絵を提示。下に候補3枚。
- **操作**：反対の意味の語を選ぶ。
- **データ**：§6 `opposites.json`。
- **DoD**：反対ペア判定・ヒント・正解演出。

#### 【L4】ことばがかわる（発展・P4）
- **ねらい**：濁点/半濁点/小文字で語の意味が変わる面白さ（`かき`→`かぎ`）。
- **画面**：語＋絵。スタンプや小文字化で別の語に変化させ、変わった絵を見せる。
- **DoD**：変化前後の語ペアデータ・変化操作・両方の絵/音。優先度低。

### Pillar E ── みじかい ぶんを よむ（読解の芽）

#### 【B1】えにあうぶん（P4）
- **ねらい**：短い文を読んで意味をとる（わかち書き）。
- **画面**：絵1枚＋短文3択（例 絵=走る犬 → `いぬが はしる`/`いぬが ねる`/`とりが とぶ`）。
- **操作**：絵に合う文を選ぶ。文は**わかち書き**（分かち書き）で表示。
- **データ**：§6 `sentences.json`（文＋正誤＋絵）。
- **DoD**：文の提示・絵とのマッチ判定・読み上げ補助。

## 5. ごほうび・進捗システム

### 5.1 星（ほし）
- 各正解で **星1**。ゲーム完走ボーナス **星+2** 程度（§10で調整可）。
- 星は「たまごゲージ」を満たす。満タンで **たまごが孵化**。

### 5.2 たまご → 孵化 → 仲間
- ゲージ満タン時、`reward` で **たまごが割れて仲間が1体出現**（未取得優先）。
- 出現後、**ずかん**に自動登録。ガオが一緒に喜ぶ。

### 5.3 ずかん
- 3系統タブ（きょうりゅう／むし／おばけ）。取得済みは絵入り、未取得はシルエット＋「？」。
- タップで **名前を音声再生＋鳴き声**。**名前表示は ひらがな/カタカナ 切替**（読みの副次学習）。
- コンプリートで特別演出（花火など）。

> ごほうびは外発的動機。学習の楽しさ（正解の気持ちよさ・しりとりの盛り上がり）を主、ごほうびを従に。星のインフレを避け、1レッスンで **たまご0〜1個**孵る程度（§10）。

## 6. 学習コンテンツ・データ設計

> **すべてJSONで外部化**。コード変更なしに文字・単語・難易度を編集できること。

### 6.1 文字データ（書き用・筆順）
- 対象：ひらがな清音46＋カタカナ清音46。**用途は主に「書き」と「対応」**（読みは習得済み前提）。
- 濁音・半濁音・拗音は §6.3 の特殊音データで扱う。

**`data/kana.json` スキーマ**
```json
{
  "hiragana": [{ "kana": "あ", "reading": "a" }, { "kana": "く", "reading": "ku" }],
  "katakana": [{ "kana": "ア", "reading": "a" }, { "kana": "ク", "reading": "ku" }]
}
```

## 7. なぞり判定アルゴリズム（W1/W2 実装詳細）

5歳向けなので**厳密さより成功体験**を優先する。

1. **前処理**：現在の画のSVG `path` を長さ計測し、`getPointAtLength` で等間隔に `N` 点サンプリング（例 `N = パス長 / 8`）。点列 `points[]` を得る。
2. **進行管理**：`cursor = 0`。指座標が `points[cursor]` から半径 `R`（既定26px、`config.trace.toleranceRadius`）以内に入り、`cursor+1` 以降へ進んでいれば `cursor++`。
3. **画クリア**：`cursor` が末尾到達で当該画を確定。次の画へ。
4. **寛容化**：一時的に線から外れても即失敗にしない（`R`内へ戻れば継続）。始点付近の当たり判定は大きめ。逆走・大外れのみ「もういちど、じゅんばんに なぞってね」を音声で促し、その画を最初から。
5. **視覚FB**：なぞった軌跡を明るく塗り、現在の画を強調、完了画は色を残す。
6. **完了**：全画クリアで文字全体を色づけ＋キラキラ＋星。
7. **禁止**：手書き文字認識は行わない。「順番通りになぞれたか」のみ判定。

### 8.3 ミニゲーム共通インターフェース（`game-base.js`）
すべてのミニゲームはこの契約を満たす。`lesson`/`game-select` はこの契約だけに依存する。
```js
export default class GameBase {
  init(ctx, services, params, question) {}  // services:{audio,speech,input,storage}
  update(dt) {}
  render(ctx) {}
  onPointerDown(x, y) {}
  onPointerMove(x, y) {}
  onPointerUp(x, y) {}
  isComplete() { return false }
  getResult() { return { correctCount: 0, mistakes: 0, itemsPracticed: [] }; }
  dispose() {}
}
```
- `lesson` の役割：§9の習得度で出題キューを作り、ゲームを順に回して結果を集計、`reward` へ渡す。

### 8.4 入力（`input.js`）
- Pointer Events で タッチ・マウス・ペン統一。Canvas表示サイズと内部解像度のスケール差を吸収し**内部座標**を渡す。マルチタッチは最初の1点のみ採用。

### 8.5 音声（`audio.js` / `speech.js`）
- **音声アンロック**：iOS/Safari等は初回操作までオーディオ不可。`title`のタップで `AudioContext.resume()`＋無音再生でアンロック（*Bramble Boom* と同手法）。
- **読み上げ（TTS）**：`speechSynthesis`、`lang="ja-JP"`、`rate ≈ 0.95`。日本語ボイスを選択。
- **端末差の注意**：ボイスの有無・品質は端末依存。**録音音声ファイルのフォールバック**（`assets/audio/voice/`）を用意し、`config.voiceMode:"tts"|"file"` で切替。既定 `tts`、将来肉声へ差し替え可能。
- **効果音**：`ピンポン` `ん?(控えめ)` `キラキラ` `孵化` 等を `assets/audio/sfx/`。BGMは静かなループ・音量控えめ・OFF可。

### 8.6 保存（`storage.js`）
- `localStorage` にJSON保存。キー例 `mojisaurus.save.v1`。`version` を持ち、破損時は初期化。
```json
{
  "version": 1,
  "stars": 12, "eggProgress": 12,
  "collection": { "dino": ["dino_01"], "insect": ["ins_03"], "ghost": [] },
  "mastery": {
    "write:し": { "attempts": 3, "cleared": 3, "level": 3 },
    "kata:シ":  { "seen": 5, "correct": 4, "level": 2 },
    "vocab:food": { "seen": 4, "correct": 4, "level": 3 }

}
```

### 8.7 保護者ゲート（`parent.js` 入口）
- 誤操作防止に**大人向けゲート**（既定：数字を順にタップ 3→1→5、または2秒長押し）。
- 突破後の `parent`：**進捗表示**（書けた字／カタカナ／特殊音／語彙の習得状況、集めた仲間数）、**設定**（BGM・効果音音量・音声モード・1レッスンのゲーム数・仲間名の表記・**データリセット**（二重確認））。

## 9. 習得度モデルと出題選定（`state/progress.js`）

軽量な間隔反復（spaced-repetition-lite）。**「文字」でなく「スキル項目(item)」単位**で管理する。

- **item の例**：`write:あ`（ひらがな書き）、`write:ア`（カタカナ書き）、`kata:シ`（まぎらわしいカタカナ）、`dakuon:が`、`yoon:きゃ`、`joshi:は`、`vocab:food`（カテゴリ語彙）、`opposite:大小` 等。
- 各itemに `attempts/correct(またはcleared)/level(0-3)` を保持。正解でlevel上昇、不正解で据え置き〜微減。`level>=3` で「できた」表示。
- **出題選定（lesson が使用）**：
  1. ホームで選ばれた**モード**（かく／ことばあそび／カタカナ・とくべつ）に対応するitem群を対象にする。
  2. 未導入itemから**新規1〜2件**を導入（カリキュラム順）。
  3. 既習のうち **level が低い（苦手）item を優先**して復習に混ぜる。
  4. 1レッスンの分量は `settings.sessionLength`（既定4ゲーム）。
- **狙い**：できないものばかり続けて自信を失わせない。得意も混ぜ、成功体験を維持。読める子が退屈しない程度に新規itemを供給する。

## 13. 実装上の注意（ハマりどころ）

- **音声アンロック**を忘れると「音が出ない」で詰まる。必ず `title` の初回タップで処理。
- **TTSの端末差**：日本語ボイスが無い端末に備え、`voiceMode:"file"` フォールバックを最初から設計に入れる。
- **座標スケール**：Canvas内部解像度とCSS表示サイズのズレで「タップがずれる」事故が起きやすい。`input.js` で必ず正規化。
- **なぞりは甘めに**：判定を厳しくすると5歳が嫌がる。まず `R` を大きく「できた！」を量産。
- **しりとりの語尾処理**：`ん`終わり・長音`ー`・小さい`っ`・拗音終わり（`きょ`→`よ`ではなく`きょ`始まり許容か等）の接続ルールを §L1 とデータで一貫させる。曖昧なら「実在する子ども向け語で自然につながる」方を優先。
- **助詞の読み上げ**：B2で`は`→「わ」、`を`→「お」、`へ`→「え」と読ませる。TTSが文脈で正しく読まない場合は、正解文に**読み仮名を付けた別テキスト**を用意してそれを読ませる。
- **否定演出を強くしない**：間違えても責めない。全ゲームで「必ず正解にたどり着ける」導線を担保。
- **KanjiVG のライセンス帰属**（CC BY-SA 3.0）を `CREDITS.md` に明記（帰属漏れ注意）。
